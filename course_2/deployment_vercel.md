# 🚀 Vercel 배포 가이드 (Course 2: 점심 메뉴 투표)

이 프로젝트는 별도의 데이터베이스 없이 구글 시트(Apps Script)를 저장소로 활용하며, 로컬 서버가 API 프록시 역할을 수행한다. 이를 무료 호스팅 서비스인 **[Vercel](https://vercel.com/)**에 배포하여 누구나 접속 가능한 환경을 구축하는 방법을 정리한다.

---

## 1. 📂 Monorepo 설정 (Root Directory)
`Antigravity` 저장소 내에는 여러 프로젝트 폴더가 존재한다. Vercel이 이 중 `course_2` 폴더를 정확히 인식하여 배포하도록 설정한다.

1. Vercel 대시보드에 접속하여 **[Add New...] > [Project]**를 생성한다.
2. 검색된 저장소 리스트에서 `Antigravity`를 찾아 **[Import]** 버튼을 클릭한다.
3. 프로젝트 설정 화면의 **Root Directory** 항목에서 **[Edit]**을 클릭한다.
4. 폴더 트리에서 **`course_2`**를 선택한 후 설정을 저장한다.

---

## 2. 🔐 환경 변수 (.env) 설정 (필수 사항)
보안 관리를 위해 `.env` 파일은 GitHub에 업로드하지 않았다. 따라서 Vercel 대시보드에서 직접 환경 변수를 주입해야 서버가 구글 시트 주소를 정상적으로 참조할 수 있다.

1. 배포 설정 화면 하단의 **Environment Variables** 섹션을 확인한다.
2. 아래의 변수 정보를 입력하고 **[Add]** 버튼을 클릭하여 등록한다.
   - **NAME**: `GOOGLE_WEB_APP_URL`
   - **VALUE**: 구글 시트에서 발급받은 웹앱 URL (따옴표를 제외한 주소만 입력)
3. 향후 주소가 변경될 경우에도 **[Settings] ❯ [Environment Variables]** 메뉴를 통해 언제든지 수정이 가능하다.

---

## 3. ⚡️ Vercel 서버리스(Serverless) 통신 구동 방식
로컬 환경에서는 `server.js`를 지속적으로 실행해야 했으나, Vercel은 다른 방식으로 구동된다.

Vercel은 프로젝트 내 **`/api` 폴더에 포함된 파일들을 자동으로 '서버리스 함수(Serverless Functions)'로 전환**한다. 투표 버튼 클릭으로 `/api/vote` 요청이 들어오는 순간에만 `api/vote.js`가 일시적으로 실행되므로 자원 관리가 매우 효율적이다.

---

## 4. 🚀 배포 완료 및 확인
1. 모든 설정을 마친 후 하단의 **[Deploy]** 버튼을 클릭한다.
2. 배포 과정이 완료되면 고유한 접속 주소(`https://...vercel.app`)가 생성된다.
3. 해당 주소로 접속하여 모바일이나 다른 기기로 투표를 진행해 본다. 구글 시트에 실시간으로 데이터가 기록되면 배포 성공이다.

🎉 이제 점심 투표 앱이 정식으로 웹상에 배포되었다. 생성된 URL을 공유하여 실제 투표를 진행할 수 있다.
