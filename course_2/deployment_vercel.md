# 🚀 Vercel 배포 가이드 (Course 2: 점심 메뉴 투표)

이 프로젝트는 별도의 데이터베이스 없이 구글 시트(Apps Script)를 저장소로 쓰고, 로컬 서버가 API 프록시 역할을 해. 이걸 무료 호스팅인 **[Vercel](https://vercel.com/)**에 배포해서 누구나 접속 가능하게 만드는 방법을 정리할게.

---

## 1. 📂 Monorepo 설정 (Root Directory)
우리 깃헙(`Antigravity`) 안에는 폴더가 여러 개 있잖아. Vercel이 이 중에서 `course_2`만 딱 골라서 배포하게 환경을 맞춰줘야 해.

1. Vercel 대시보드 들어가서 **[Add New...] > [Project]** 하나 만들어.
2. 내 깃헙 저장소 중에서 `Antigravity` 찾아서 **[Import]** 눌러.
3. 프로젝트 설정 화면 중간에 **Root Directory**라는 메뉴가 보일 거야. 거기 **[Edit]** 눌러서 **`course_2`** 폴더를 선택하고 저장해.

---

## 2. 🔐 환경 변수 (.env) 설정 (제일 중요!)
깃헙에는 보안 때문에 `.env` 파일을 안 올렸어. 그래서 Vercel 대시보드에서 직접 이 값을 넣어줘야 서버가 구글 시트 주소를 알아챌 수 있어.

1. 배포 설정 화면 하단에 **Environment Variables**라는 섹션 찾아봐.
2. 아래 내용을 그대로 입력하고 **[Add]** 눌러서 등록해.
   - **NAME**: `GOOGLE_WEB_APP_URL`
   - **VALUE**: 네가 구글 시트에서 복사한 그 긴 웹앱 URL (따옴표 빼고 주소만!)
3. 나중에 주소가 바뀌어도 프로젝트 설정의 **[Settings] ❯ [Environment Variables]** 들어가면 언제든 바꿀 수 있어.

---

## 3. ⚡️ Vercel 서버리스(Serverless) 알아두기
로컬에서 돌릴 때는 `server.js`를 계속 켜놨어야 했지? 

Vercel은 좀 달라. 프로젝트 안에 **`/api` 폴더가 있으면 그 안의 파일들을 자동으로 '서버리스 함수'라는 걸로 바꿔줘.** 
누군가 투표를 클릭해서 `/api/vote`로 신호가 들어올 때만 `api/vote.js`가 잠깐 실행되는 방식이라 훨씬 빠르고 관리할 게 없어서 편해!

---

## 4. 🚀 배포 끝! 확인하기
1. 설정 다 했으면 맨 아래 **[Deploy]** 버튼 눌러.
2. 폭죽 터지면서 배포 완료되면 주소가 하나 나올 거야. (`https://어쩌구.vercel.app`)
3. 그 주소로 들어가서 폰이나 다른 컴퓨터로 투표해봐. 구글 시트에 실시간으로 꽂히면 성공이야!
